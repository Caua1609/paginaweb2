O que é DCL?

No âmbito da Linguagem de Consulta Estruturada (SQL), a Linguagem de Controle de Dados (DCL) desempenha um papel fundamental na administração de bancos de dados, gerenciando os direitos de acesso e as permissões dos usuários. Esses comandos são os guardiões do banco de dados, garantindo que os usuários tenham acesso apenas aos dados que estão autorizados a visualizar e manipular. Os três comandos principais que constituem a DCL são GRANT, DENY e REVOKE

GRANT

Este comando é usado para atribuir privilégios específicos a usuários ou funções, permitindo que executem determinadas ações em objetos do banco de dados Exemplo A equipe da recepção precisa visualizar os dados dos pacientes e agendar novas consultas. No entanto, eles não devem ter acesso aos diagnósticos e prescrições.

Exemplo de Código SQL

A primeira instrução GRANT permite que o usuário recepcionista execute o comando SELECT em todos os campos da tabela Pacientes.

A segunda instrução é mais restritiva: ela concede a permissão de INSERT apenas nas colunas id_paciente, id_medico e data_consulta da tabela Consultas. Além disso, permite que o recepcionista visualize (SELECT) apenas essas mesmas colunas, protegendo as informações sensíveis de diagnostico e prescricao.

REVOKE

O comando REVOKE é o oposto do GRANT. Ele remove permissões que foram previamente concedidas a um usuário.

DENY

O comando DENY é usado para negar explicitamente uma permissão a um usuário. A principal diferença entre REVOKE e DENY é que o DENY tem precedência sobre as permissões concedidas. Se um usuário pertence a um grupo que tem uma permissão GRANT, mas o usuário individualmente tem um DENY para a mesma permissão, o acesso será negado.

As funções agregadas em SQL servem para resumir ou consolidar informações em vez de mostrar cada linha individualmente, quais são elas?

AVG

Calcula a média dos valores de uma coluna numérica.

COUNT

Faz a contagem de linhas (ou valores não nulos de uma coluna).

MAX

Retorna o maior valor de uma coluna.

MIN

Retorna o menor valor de uma coluna.

SUM

Faz a soma de todos os valores de uma coluna numérica.

DDL

É uma parte fundamental da linguagem SQL, usada para definir, modificar e apagar a estrutura de um banco de dados.

CREATE:

O comando CREATE é usado para criar novos objetos em um banco de dados, como tabelas, índices, visualizações (views) e esquemas (schemas).

DROP

O comando DROP é usado para apagar objetos de um banco de dados. Use com extrema cautela, pois a operação é irreversível e apaga o objeto e todos os dados contidos nele

ALTER

O comando ALTER é usado para modificar a estrutura de um objeto de banco de dados que já existe.

DML

Data Manipulation Language (DML ) - A Data Manipulation Language é o sub idioma responsável pela adição, edição ou exclusão de dados de um banco de dados. A Linguagem de Manipulação de Dados, ou DML , é o grupo de comandos responsáveis pela manipulação de dados em um banco de dados; isto geralmente implica em inserir, editar ou excluir linhas em tabelas SQL.

MODELO DE BANCO DE DADOS

INSERT

Usado para adicionar novos registros a uma tabela. A palavra-chave INTO é usada para deixar claro que você está direcionando a operação de inserção para uma tabela específica. O INTO é só uma palavra que ajuda a mostrar que você está falando sobre a tabela onde os dados vão entrar. É como dizer "colocar na tabela".

UPDATE

Usado para modificar registros existentes em uma tabela

DELETE

Usado para remover registros de uma tabela

SELECT

Usado para consultar e recuperar dados de uma ou mais tabelas.
